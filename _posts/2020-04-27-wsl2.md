# WSL2를 위한 Window 10 build >= 18917 빌드
Microsoft 문서에 따르면 WSL2를 사용하기 위한 조건으로 아래와 같이 설명합니다.

---
> WSL 2는 Windows 10 빌드 18917 이상에서만 사용할 수 있습니다.

- WSL이 설치되어 있는지 확인하고(관련 지침은 여기 참조) Windows 10 빌드 18917 이상을 실행 중인지 확인
    - 빌드 18917 이상을 사용 중인지 확인하려면 [Windows 참가자 프로그램](https://insider.windows.com/en-us/)에 참여하고 '초기' 링 또는 '이후' 링을 선택하세요.
    - 명령 프롬프트를 열고 ver 명령을 실행하여 Windows 버전을 확인할 수 있습니다.
- '가상 머신 플랫폼' 옵션 구성 요소 사용
- 명령줄을 사용하여 WSL 2에 의해 지원되도록 Distro 설정
- Distro가 사용 중인 WSL 버전 확인

---
위를 위해 Windows 10을 18917 이상의 버전으로 빌드업 해야하는데... 이는 윈도우의 참가자 프로그램을 통해 20H1, 20H2 프리뷰 업데이트를 진행해야 사용할 수 있습니다.

# Update WSL2
자, WSL2로 업데이트해봅시다.

WSL은 `Docker for Windows`를 사용하지 않고 `native Docker`를 사용할 수 있는 장점이 있어 실제 linux를 이용하는 것과 동일하게 코딩할 수 있다고 합니다!
- `WSL2`에서는 `nvidia-docker`를 사용할 수 있다는 링크를 본 기억이 있는데 이는 향후 추가적으로 업로드하겠습니다!

### 'Virtual Machine 플랫폼' 옵션 구성 요소를 사용하고 WSL이 사용하도록 설정되었는지 확인
우선 window `PowerShell`을 **"관리자"** 로 실행하시고 아래 두 명령어를 입력합시다.
```
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

위 명령어를 입력 후 `다시 시작`하시고,

### 명령줄을 사용하여 WSL 2에 의해 지원되도록 Distro 설정
`WSL2`로 버전업을 하기 위해 아래와 같이 입력합시다.
```
wsl --set-version <Distro> 2
```
위 `<Distro>`엔 Distro의 실제 이름으로 대체하라는 데, 아래 명령어로 확인할 수 있습니다. 제 PC는 `Ubuntu-18.04`네요. (위에서 그렇게 설치했으니까요 :)
```
PS C:\WINDOWS\system32> wsl -l
Linux용 Windows 하위 시스템 배포:
Ubuntu-18.04(기본값)
```

즉, 제 경우엔 이렇게 명령어를 치면 되겠죠?
```
wsl --set-version Ubuntu-18.04 2
```

아래 명령어로 Distro가 사용 중인 WSL의 버전을 확인해 봅시다.
```
PS C:\WINDOWS\system32> wsl --list --verbose
  NAME            STATE           VERSION
* Ubuntu-18.04    Running         2

# 같은 명령어입니다.
PS C:\WINDOWS\system32> wsl -l -v
  NAME            STATE           VERSION
* Ubuntu-18.04    Running         2
```

### 만일, 아래와 같은 에러가 발생한다면?
```
WSL 2에 커널 구성 요소 업데이트가 필요합니다.
자세한 내용은 https://aka.ms/wsl2kernel을 참조하십시오.
```
위 링크에 들어가셔서 파일을 다운로드하시면 해결됩니다 :)

### Troubleshooting Err?
- PS C:\WINDOWS\system32> wsl --set-version Ubuntu-16.04 2
가상 디스크 시스템 제한으로 인해 요청한 작업을 완료할 수 없습니다. 가상 하드 디스크 파일은 압축이 풀려 있는 상태이고 암 호화되지 않아야 하며 스파스가 아니어야 합니다.

위 링크에서는 `curl -o- <link> | bash`로 설치하라고 나오는데 저는 에러가 떠서 6번 링크를 참고하여 아래와 같이 작성했습니다.


### Docker 설치하기
개인적으로 WSL을 사용하려했던 이유가 Docker였는데 이전 WSL에서는 Docker Daemon을 읽지 못하는 에러가 계속 발생하여 저는 `WSL2`로 버전업을 진행하게 됐어요!

해당 블로그에서 위 이유에 대해 아래와 같이 설명하고 있더군요.
> 기본적을 WSL은 docker 데몬을 지원하지 않는다.

Native Docker를 설치하기 위해 아래의 명령어를 입력해주세요.

```
$ sudo apt-get install \
    apt-transport-https \
    ca-certificates \
    curl \
    software-properties-common
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
$ sudo apt-key fingerprint 0EBFCD88
$ sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
$ sudo apt-get update
$ sudo apt-get install docker-ce
```

설치가 완료되면 docker service를 root 권한으로 실행합니다.
- 추가하자면, `sudo`로 명령어를 사용하는 것이 root 권한으로 실행하는 것이며
- `service`는 `systemctl`을 wsl에서 사용하는 것이라 생각하시면 됩니다!

```
$ sudo service docker start
 * Starting Docker: docker
$ sudo service docker status
 * Docker is running
```

docker 명령어를 root 권한이 아닌 사용자 권한으로 실행시키기 위해 사용자를 docker 그룹에 추가합시다!
```
$ whoami
jinmang2 # WSL을 설치할 때 입력한 Username입니다.
$ sudo usermod -aG docker $(whoami)
# 터미널 종료 후 다시 열어주세요!
$ exit
```

잘 되는지 확인해봅시다.
```
$ docker images
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
$ docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS              PORTS               NAMES
```

정상적으로 잘 되는 군요 :)

# Linux에 conda 설치하기
10과 11번 링크 참고.
[https://repo.continuum.io/archive](https://repo.continuum.io/archive)에서 Linux에 맞는 버전의 shell파일의 이름을 복사해주세요!!
저는 `Anaconda3-2020.02-Linux-x86_64.sh`로 설치했습니다.

우선, 관리자로 Shell을 실행했다면 아래 디렉토리로 설정되어 있을거에요.
```
(base) [USERNAME]@DESKTOP-0G7CR43:/mnt/c/WINDOWS/system32$
```

이를 `/home/[USERNAME]`으로 바꿔줍시다.
```
(base) [USERNAME]@DESKTOP-0G7CR43:/mnt/c/WINDOWS/system32$ cd /home/jinmang2
(base) jinmang2@DESKTOP-0G7CR43:/ $
```

아래 커맨드로 파일을 다운로드 받고
```
wget https://repo.continuum.io/archive/[파일명]
wget https://repo.continuum.io/archive/Anaconda3-2020.02-Linux-x86_64.sh
```

모두 내려받았다면 아래 명령어로 Anaconda3 설치를 진행합시다.
```
bash [파일명]
bash Anaconda3-2020.02-Linux-x86_64.sh
```

11번 링크에 나오는 대로, 환영 메세지, 라이센서와 관련된 메세지 등을 전부 Enter / yes로 진행하고 설치가 완료될 때 까지 기다립시다.

설치가 완료되면 아래 명령으로 conda를 실행하면 우선 anaconda 설치는 완료입니다.
```
source ~/.bashrc
```

# Conda 가상환경 만들기
이제부턴 window에서 했던 것과 동일합니다.
```
$ conda update conda
$ conda create -n [가상환경이름] numpy pandas matplotlib python=3.6 # 각자 버전에 맞게.
$ conda info --envs # 가상 환경 리스트 보기
$ conda activate [가상환경이름] # 가상 환경 실행
$ conda deactivate # 가상 환경 해제
```
